Overview

simple_docker is an Eiffel library for programmatic Docker container management. It provides a clean, type-safe API for building, running, and managing Docker containers through Windows named pipes, communicating directly with the Docker Engine API.

Part of the Simple Eiffel ecosystem.

Quick Start

Installation

  1. Set the environment variable: 
    export SIMPLE_DOCKER=/path/to/simple_docker
  2. Add to your ECF file: 
    <library name="simple_docker" location="$SIMPLE_DOCKER/simple_docker.ecf"/>

Basic Usage

local
    client: DOCKER_CLIENT
    spec: CONTAINER_SPEC
do
    create client.make

    if client.ping then
        -- Docker daemon is responsive
        create spec.make ("alpine:latest")
        spec.set_name ("my-container")
            .set_cmd (<<"echo", "Hello from Eiffel!">>)
            .add_env ("FOO", "bar").do_nothing

        if attached client.run_container (spec) as c then
            print ("Created: " + c.short_id + "%N")
        end
    end
end

Key Features

Container Lifecycle

Create, start, stop, pause, restart, kill, and remove containers with full state tracking.

Image Management

List, pull, inspect, and remove Docker images from local and remote registries.

Fluent Builder API

Configure containers using intuitive chained method calls with full type safety.

Design by Contract

Full DBC support with preconditions, postconditions, and class invariants.

Named Pipe Transport

Direct communication with Docker Engine via Windows named pipes.

Chunked Transfer

Proper HTTP/1.1 chunked transfer encoding for reliable API communication.

API Reference

DOCKER_CLIENT

Main facade for all Docker operations.

Feature Description
ping Check if Docker daemon is responsive
version Get Docker version information
info Get Docker system information
list_containers (all) List containers (optionally include stopped)
create_container (spec) Create container from specification
run_container (spec) Create and start container
start_container (id) Start a stopped container
stop_container (id, timeout) Stop container with timeout
remove_container (id, force) Remove container
container_logs (id, stdout, stderr, tail) Get container logs
wait_container (id) Wait for container to exit
list_images List all images
pull_image (name) Pull image from registry

CONTAINER_SPEC

Fluent builder for container configuration.

Feature Description
set_name (name) Set container name
set_hostname (hostname) Set container hostname
set_cmd (args) Set command to run
add_port (container, host) Map TCP port
add_env (key, value) Add environment variable
add_volume (host, container) Mount volume
set_memory_limit (bytes) Set memory limit
set_restart_policy (policy) Set restart policy
to_json Export as JSON for Docker API

DOCKER_CONTAINER

Container representation with state queries.

Feature Description
id / short_id Container ID (full/short)
name Container name
is_running Check if container is running
is_exited Check if container has exited
can_start Can container be started?
exit_code Exit code (if exited)
has_exited_successfully Did container exit with code 0?

DOCKER_IMAGE

Image representation with metadata.

Feature Description
id / short_id Image ID (full/short)
repo_tags List of repository tags
size / size_mb Image size in bytes/MB
has_tag (tag) Check for specific tag
matches (ref) Match by name or ID

Examples

Web Server Container

create spec.make ("nginx:alpine")
spec.set_name ("web-server")
    .add_port (80, 8080)
    .add_volume ("/var/www", "/usr/share/nginx/html")
    .set_restart_policy ("unless-stopped").do_nothing

container := client.run_container (spec)

Container State Queries

if container.is_running then
    print ("Container is running%N")
elseif container.is_exited then
    if container.has_exited_successfully then
        print ("Exited with code 0%N")
    else
        print ("Exited with code: " + container.exit_code.out + "%N")
    end
end

-- Check what operations are allowed
if container.can_start then
    client.start_container (container.id).do_nothing
end

Error Handling

container := client.create_container (spec)

if client.has_error then
    if attached client.last_error as err then
        if err.is_not_found then
            -- Image not found, pull it
            client.pull_image (spec.image).do_nothing
        elseif err.is_conflict then
            -- Container name exists
        elseif err.is_retryable then
            -- Network error, can retry
        end
    end
end

Listing and Filtering

-- List running containers
across client.list_containers (False) as c loop
    if c.is_running then
        print (c.name + " is running%N")
    end
end

-- Find image by tag
across client.list_images as img loop
    if img.has_tag ("alpine") then
        print ("Found alpine: " + img.short_id + "%N")
    end
end
